PPT - Splunk Platform

Splunk software transforms your machine data into valuable real-time insights. In the first part of our demo, we will examine various examples of machine data and learn how Splunk helps you to quickly extract value out of these logs.

Is this a representative example of typical logs in your organization? What would you add?

The challenge with machine data is that it is unstructured. Each one of these logs looks very different from each other. Before Splunk, getting value out of this data took months of expensive transformation to put it into some type of structure for analysis.

Let's take a look at how Splunk organizes data.

Click on # in CRM box.

This is a search of SplunkTel's CRM or business logs, which is one example of the machine data available for analysis.  Don't worry too much about the ugly details of these raw logs. I'm going to help you see how easy it is to use this interface and get value from these logs. Let me explain this basic search interface.

Point to search bar

In this search bar, you can add "google" like search strings. In this case, we are telling the system to look for a very specific type of event, but I could have easily just searched using the word "error" or "iphone".

Point to time range picker and then histogram

Next we tell Splunk what time range we are searching across. As data is returned, Splunk will build a histogram from matching events. Immediately we can see some sort of anomaly here.

Point to raw events

This panel provides me with the raw event details. Regardless of the data format, we can always capture the raw event.

Point to fields on left

What you see here is the secret sauce for Splunk, which is the ability to extract knowledge at search time from the raw events. We call this "Schema on the fly". This is important because Splunk can dynamically change as your data changes, without having to go back and re-architect your datastore therefore saving you time and money. In addition to key value pairs, Splunk also understands structured data such as json, csv (tsv, psv, etc), or XML. If events lack a self-describing structure, Splunk can be configured to extract structure by using Regular Expression based extractions. Many of the common data types have been configured for you and are available for free from Splunk's website.

So far we have been looking at one log type, let's expand our search to include our authentication logs with our business events.

Select Searches & Reporting => OI Music Service (click on search icon )

Point to the radius event

This first event, comes from the radius authentication system. It provides information such as MDN or phone number, IP address assigned, and when it is a billing initiation (start) or billing complete (stop). The second event, is from the web tier and it provides information about what IP address searched for or downloaded a song. What I really want to know is what phone number searched for or downloaded a song. To get this, I need to combine these two events.

Let's add to our search, Splunk's powerful transaction command.

Select Searches & Reporting => OI Music Service Transaction (click on search icon

 )

This has combined my radius and web events into a single event with all the necessary information for reporting.

For many organizations, this default search interface is light years ahead of the tools they currently use such as manually searching through logs from the command line or writing custom scripts to pull out information file by file. With Splunk we quickly and easily pull this information together and provide an easy to use interface to extract value.
